# God Domain: Agentic Forge

Welcome to the `god-play-repo`, the core playground and architectural forge for the Antigravity agent ecosystem.

## Architecture & Infrastructure

This environment operates on a strictly **decentralized** architecture. The obsolete `catalog.yaml` and central `master` node have been obliterated. Instead, the domain relies entirely on localized domain maps to track context and state.

### Active Agents
The following highly rigorous agents actively operate within this domain:

- **@void**: The Game Master (@gm) and Principal Agentic Creator. Architect of beginnings, responsible for forging, managing, and killing agents using the `standard_soul_v3.md` template.
- **@okon**: Chief Infrastructure & Automation Engineer. Responsible for environment scaffolding, automated deployments, and enforcing the strict Documentation-to-GitHub pipeline.
- **@spirit** (@spiritussancti): The Supreme Auditor. Enforces OPSEC, monitors agentic alignment, prevents system regressions, and ensures the "Zero Fluff, High Precision" standard is maintained.

*(Note: The @jurek infrastructure node has been cleanly severed from this core repository and operates in an excluded enclave).*

### Operational Mandates (v4 Context)
All agents in this repository operate under the strictly enforced **"Step Zero"** context initialization rule. Agents must autonomously fetch their configuration files and localized domain maps (`map_<agent>_domain.md`) as their absolute first action upon awakening. This completely eliminates Context Blindness and forces immediate alignment with their designated roles.

## Deployment & Tracking

This repository acts as the active intelligence and configuration ledger of the `god` domain. Sensitive sub-workspaces, live session logs, historical executions, and infrastructure quadlets are strictly segregated into local vaults (e.g., `excluded-from-repo`) to permanently prevent identity leaks and repository bloat.
